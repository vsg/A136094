Solver for OEIS A136094 sequence. Finds smallest numbers containing all permutations of digits {1,...,n} as subsequences.

Usage: solve [<profile>] [options]

Profiles:

  7             Alias for 7-dfs-batch
  7-bfs         n=7, simple single-threaded BFS (precalc using DFS)
  7-dfs         n=7, simple single-threaded DFS (precalc using DFS)
  7-dfs-batch   n=7, multi-threaded DFS (precalc using DFS)
  7-dfs-swarm   n=7, multi-threaded hybrid DFS/DFS-batch

  8             Alias for 8-dfs-batch
  8-dfs-batch   n=8, multi-threaded DFS (precalc using DFS)
  8-dfs-disk    n=8, multi-threaded DFS (precalc using DFS)
  8-dfs-swarm   n=8, multi-threaded hybrid DFS/DFS-batch

  9-precalc7    n=9, precalc using DFS up to size 7 
  9-precalc9    n=9, precalc using DFS-batch up to size 9
  9-solve       n=9, solve using DFS-disk

Algorithms:

  bfs           Single-threaded breadth-first search
  bfs-batch     Multi-threaded breadth-first search over batches of nodes
  dfs           Single-threaded depth-first search using recursion
  dfs-loop      Single-threaded depth-first search using loop
  dfs-batch     Multi-threaded depth-first search over batches of nodes
  dfs-swarm     Multi-threaded hybrid: use "dfs" per problem, switch to "dfs-batch" when few problems left
  dfs-disk      Multi-threaded depth-first search over bigger blocks of nodes, progress saved to disk

Options for bfs-batch, dfs-batch:
  --node-batch-size=<int>         Batch size (default: 10 000)

Options for dfs-batch:
  --dfs-batch-max-cache=<int>     Maximum size of duplicate node lookup cache (default: 250 000 000)

Options for dfs-swarm:
  --dfs-swarm-max-groups=<int>    Maximum number of "dfs-batch" solver threads (default: 2)
  --dfs-swarm-batch-mode=<bool>   Start in "dfs-batch" mode (default: false)

Options for dfs-disk:
  --dfs-disk-block-size=<int>     Writer block size (default: 50 000 000)
  --dfs-disk-batch-size=<int>     Reader batch size (default: 10 000 000)
  --dfs-disk-seen-size=<int>      Maximum size of duplicate node lookup cache (default: 50 000 000)

General options:

  --n=<int>                       Specify problem size, N (default: 7)
  --thread-count=<int>            Thread count (default: <all-cpu-cores>)
  --precalc-alg=<string>          Algorithm to use for subproblem pre-calculation (default: dfs)
  --solve-alg=<string>            Algorithm to use for solving the main problem (default: dfs-batch)
  --save-files=<bool>             Cache subproblems and solutions in files on disk (default: false)
  --checkpoint-shapes=<string>    List of shape sizes, for which to write checkpoint on disk (default: <none>)
  --max-precalc-shape=<string>    Stop pre-calculation after this subproblem size (default: <none>)
  --min-piece-check-size=<int>    Try splitting items above this size during subproblem lookup (default: n-2)
  --max-loop-123=<int>            Limit known subproblem lookup loop size for 1-st, 2-nd, 3-rd nested loops
  --max-loop-45=<int>             Limit known subproblem lookup loop size for 4-th, 5-th nested loops
  --apply-partials=<bool>         Add subproblems to lookup trees (default: true)
  --print-progress=<bool>         Print progress messages (default: true)
  --debug=<bool>                  Print subproblem lookup progress and results (default: false)
  --debug-next-moves=<bool>       Print candidate next moves during subproblem lookup (default: false)

Commands:

  --solve <problem>               Solve specified problem
  --help                          Show this help message

Examples:

  solve 7
  solve 7-dfs-batch
  solve 8-dfs-batch --dfs-batch-max-cache=1000000000
  solve 8-dfs-swarm
  solve 8-dfs-swarm --dfs-swarm-max-groups=4 --dfs-batch-max-cache=4000000000
  solve 7-bfs --save-files --checkpoint-shapes=1/7 --solve "1/123456 24/12347"
  solve 7-dfs --save-files --checkpoint-shapes=1/6 --max-precalc-shape=1/6 --debug --solve "1/123456 24/12347"
  solve 7-dfs --save-files --checkpoint-shapes=1/6 --max-precalc-shape=1/6 --debug --debug-next-moves --solve "1/123456 24/12347"
